# Post-processing

This folder contains the spatial post-processing script used to refine the annual raster classification maps and convert them into vector datasets for CAMAF-2019-2025.

The main script is `postprocess_refine_vectorize.py`.

## Purpose

The post-processing workflow performs the following steps:

1. Applies a coastline-based land mask to remove landward false positives.
2. Removes small isolated patches using class-specific connected-component filtering.
3. Fills small internal holes within aquaculture patches.
4. Applies light morphological closing to selected classes.
5. Converts refined raster classification maps into vector polygons.
6. Assigns class attributes to the exported vector files.

## Input data

The main inputs are:

- Annual raster classification maps generated by the TransUNet inference workflow.
- A manually prepared coastline-derived land or sea polygon mask.
- Raster metadata, including spatial resolution, coordinate reference system, and spatial extent.

The raster classification maps use the following class IDs:

| Pixel value | Description |
|---|---|
| 0 | Background |
| 1 | Northern Rafts |
| 2 | Cages |
| 3 | Southern Rafts |
| 255 | Nodata, when available |

## Land masking

A coastline-based spatial mask is used to remove landward misclassifications. The coastline-derived polygon vector is rasterized to match the resolution, extent, and coordinate reference system of each annual classification raster.

If `MASK_VECTOR_MODE = "land"`, pixels located within the landward mask are assigned to the background class. If `MASK_VECTOR_MODE = "sea"`, only pixels located within the sea mask are retained.

## Small-patch removal

Small isolated patches are removed using 8-neighbour connected-component analysis. The class-specific minimum mapping-unit thresholds are:

| Class | Threshold | Area at 10 m resolution |
|---|---:|---:|
| Northern Rafts | 30 pixels | 3,000 m² |
| Cages | 15 pixels | 1,500 m² |
| Southern Rafts | 8 pixels | 800 m² |

## Hole filling

Small internal holes are filled using class-specific thresholds:

| Class | Threshold | Area at 10 m resolution |
|---|---:|---:|
| Northern Rafts | 100 pixels | 10,000 m² |
| Cages | 50 pixels | 5,000 m² |
| Southern Rafts | 30 pixels | 3,000 m² |

A light 3 × 3 morphological closing operation is applied to Northern Rafts and Cages to improve local patch continuity. No closing operation is applied to Southern Rafts to avoid excessive expansion of fragmented coastal patches.

## Vectorization

After spatial refinement, the raster classification maps are converted to vector polygons. Only pixels belonging to the three aquaculture facility classes are exported. Background and nodata pixels are excluded.

The output attribute table contains:

| Field | Description |
|---|---|
| `gridcode` | Class code. 1 = Northern Rafts; 2 = Cages; 3 = Southern Rafts |
| `class_name` | Abbreviated class label: N-fra, CA, or S-fra |
| `year` | Mapping year |

## Output

The outputs include:

- Refined single-band GeoTIFF classification maps.
- Annual vector polygon files.

By default, the script exports vector data as ESRI Shapefiles to match the released CAMAF-2019-2025 dataset. The output vector files include the standard `.shp`, `.shx`, `.dbf`, `.prj`, and `.cpg` components.

## User settings

Before running the script, users should modify the following parameters in `postprocess_refine_vectorize.py`:

| Parameter | Description |
|---|---|
| `YEAR` | Mapping year of the input classification map |
| `INPUT_MASK_DIR` | Folder containing annual prediction mask GeoTIFF files |
| `OUTPUT_RASTER_DIR` | Folder for refined raster outputs |
| `OUTPUT_VECTOR_DIR` | Folder for vector outputs |
| `COAST_VECTOR_PATH` | Path to the coastline-derived land or sea polygon mask |
| `MASK_VECTOR_MODE` | `"land"` if polygons represent land areas to remove, or `"sea"` if polygons represent sea areas to retain |
| `VECTOR_FORMAT` | Output vector format, either `"ESRI Shapefile"` or `"GPKG"` |

## Notes

The class-specific filtering and hole-filling thresholds were empirically determined through visual inspection of representative coastal regions and were kept constant for all annual maps.

The coastline input should be polygon data rather than a simple coastline line. If only coastline lines are available, they should first be converted into land or sea polygon masks before running the script.
